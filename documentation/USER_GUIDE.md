# 📘 RAG Vector 사용자 가이드

RAG Vector 시스템을 효과적으로 사용하기 위한 운영 및 사용 가이드입니다.

---

## 1. 시스템 접속 및 로그인

설치가 완료되면 브라우저를 통해 다음 서비스에 접속할 수 있습니다. 모든 관리 기능은 로그인이 필요합니다.

- **기본 계정**: `admin@example.com` / `admin123`

### 1.1 서비스 주소
- **사용자 채팅 UI**: [http://localhost:3000](http://localhost:3000)
- **문서 관리자 대시보드**: [http://localhost:8501](http://localhost:8501)

---

## 2. 문서 관리 (운영자)

### 2.1 로그인 및 업로드
1. 관리자 대시보드 접속 후 제공된 계정으로 로그인합니다.
2. **"문서 업로드"** 탭으로 이동합니다.
2. **"파일 업로드"** 영역에 문서를 드래그하거나 파일을 선택합니다. (지원 포맷: PDF, HWP, DOCX, XLSX 등)
3. **[업로드 및 처리 시작]** 버튼을 클릭합니다.
4. 하단 테이블에서 문서의 상태가 `ready`가 되면 질문할 준비가 끝난 것입니다.

### 2.2 문서 삭제 및 관리
- 잘못 업로드된 문서는 목록 옆의 **[삭제]** 버튼을 통해 제거할 수 있습니다.
- 문서를 삭제하면 연관된 모든 벡터 데이터(지식)가 함께 제거됩니다.

---

## 3. AI와 대화하기 (사용자)

구축된 지식을 바탕으로 질문을 주고받는 단계입니다.

### 3.1 질문 작성 팁
- **구체적으로 질문하기**: "휴가 규정 알려줘" 보다는 "2년차 직원의 연차 개수와 신청 절차가 어떻게 돼?"와 같이 구체적으로 물을 때 더 정확한 답변을 얻을 수 있습니다.
- **키워드 포함하기**: 우리 시스템은 키워드 매칭을 가중치로 사용하므로, 전문 용어나 고유 명사를 포함하면 검색 정확도가 올라갑니다.

### 3.2 출처 확인
- AI의 답변 하단에는 답변의 근거가 된 **참조 문서 목록**이 표시됩니다.
- 실제 답변 내용이 정확한지 문서 원본의 내용과 비교하며 확인할 수 있습니다.

---

## 4. 관리 및 모니터링

### 4.1 시스템 로그 확인
시스템에 문제가 발생한 경우, Docker 로그를 통해 원인을 파악할 수 있습니다.
```bash
# 백엔드 로그 실시간 확인
docker-compose logs -f backend
```

### 4.2 성능 튜닝
답변의 품질이 만족스럽지 않은 경우, `.env` 파일의 다음 설정을 검토하세요.
- `SEARCH_K`: 답변에 참고할 문서 개수를 늘려보세요.
- `CHUNK_SIZE`: 문서의 문맥이 잘린다면 청크 크기를 조정해 보세요.
